Intent Engine — Unified Schema

6-level hierarchy with full lifecycle tracking

intents table

Ecto Schema: Ema.Intents.Intent
── Identity ──
id string "int_<ts>_<rand>"
title string required
description text optional
slug string unique, auto-generated

── Hierarchy ──
level integer 0-5
0=vision 1=goal 2=project 3=feature 4=task 5=execution
kind string enum
goal | question | task | exploration | fix | audit | system
parent_id FK → intents (self-ref)

── Lifecycle ──
status string enum
planned | active | researched | outlined | implementing | complete | blocked | archived
phase integer 1-5 (research→harvest)
clarity float 0.0-1.0
energy float 0.0-1.0
completion_pct integer 0-100

── Linkage (all optional) ──
project_id FK → projects
goal_id FK → goals
task_id FK → tasks
proposal_id FK → proposals
execution_id FK → executions
brain_dump_item_id FK → inbox_items
harvested_intent_id FK → harvested_intents
session_id FK → claude_sessions

── Content ──
wiki_path string vault note path
intent_folder_path string .superman/intents/<slug>
embedding binary vector (optional)
tags text JSON array
metadata text JSON map
source_fingerprint string unique (dedup)

── Timestamps ──
inserted_at, updated_at

intent_edges table

Ecto Schema: Ema.Intents.IntentEdge
id string
source_id FK → intents (cascade)
target_id FK → intents (cascade)
edge_type string enum
hierarchy | depends-on | blocks | enables | supersedes
implements | related-to | part-of | contradicts
weight float 0.0-1.0 (ranked relationships)
inserted_at, updated_at

Population Sources → intents

Auto-Population Pipeline (PubSub Subscribers)
brain_dump:item_created
→ create intent (level: 4-5, kind: from content classification)
→ create IntentFolder on disk
→ create execution (auto-dispatch)

proposals:approved
→ create intent (level: 2-3, kind: task)
→ link to proposal_id
→ link parent intent if cluster has intent_node_id

executions:completed
→ update linked intent: phase++, completion_pct, status
→ write to IntentFolder status.json
→ propagate status up to parent (Superman pattern)

intention_farmer:intent_loaded
→ create intent (level: 4-5, kind: from harvested type)
→ link to harvested_intent_id + session_id
→ check for cluster merge (embedding similarity)

goals:created / goals:updated
→ create/update intent (level: 1, kind: goal)
→ link to goal_id

intents:status_changed (self-referential)
→ reflexive capture: emit pipe event
→ update wiki_path note if vault convergence active
