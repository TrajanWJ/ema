## 1. What EMA Is

Personal AI desktop app: autonomous thinking companion and life OS for a solo developer managing multiple concurrent projects (ProSlync, Superman IDE, client sites, Truks dispatch).

**Stack:** Elixir/Phoenix 1.8 daemon (localhost:4488) + Tauri 2 + React 19 + Zustand + SQLite  
**Core problem solved:** Context collapse — too many tools, tabs, no single surface understanding the full picture.

## 4. Test Health

**545 tests, 3 failures** — all in `SecondBrain.Indexer`

**Root cause:** SQLite connection ownership race condition. `Indexer.index_note/1` spawns `Task.Supervised` processes that hold DB connections, but the parent process (test sandbox owner) exits before the tasks complete. The tasks then try to use connections owned by dead processes.

This is a classic Ecto sandbox + async task issue. The indexer needs either:
- Explicit `Ecto.Adapters.SQL.Sandbox.allow/3` for spawned tasks
- Or restructured to not use Task.async inside transactions

**Additionally:** Multiple `[error] Exqlite.Connection disconnected` logs during tests suggest this race condition could manifest in production under load.

## 5. Uncommitted Changes (Working Tree)

15 modified files + 1 new test file, all focused on **babysitter chatter suppression**:

### What changed and why
The babysitter stream-of-consciousness system was posting to Discord far too aggressively (tick intervals as low as 5ms-180ms). The uncommitted changes:

1. **Relaxed tick cadences dramatically** — all streams moved from millisecond auto-mode to manual-mode with 60s-3600s intervals
2. **Added state-change-driven delivery** — identical steady-state summaries are now skipped, repeated degraded summaries suppressed
3. **Added cooldown-based anomaly dispatch** — alerts cooldown 10min, live posts 2min, prevents duplicate alerts unless severity escalates
4. **Batched stalled session alerts** — groups by project, posts once with samples instead of per-session spam
5. **Softened startup** — missing DISCORD_BOT_TOKEN now warns instead of crashing (reverts commit 0f037a4)
6. **Cleaned dead code** — removed unused aliases, functions, and unreachable branches across 6 files

**These changes are coherent and intentional.** They represent post-MVP hardening to make the babysitter livable rather than noisy.

## 6. Critical System Issues

### RED — Broken / Silently Failing

**R1. AI Bridge silently bypassed**
- Config says `ai_backend: :bridge` but Bridge GenServer falls back to `Runner.run/2` when unavailable
- All Bridge features (multi-account rotation, provider failover, cost tracking, governance audit) are dead code paths
- Every AI call in the system goes through the basic CLI runner

**R2. OpenClaw streaming adapter calls undefined function**
- `Ema.Claude.Adapters.OpenClaw.stream/4` calls `ClaudeCli.stream/4` which doesn't exist
- Any streaming path through OpenClaw will crash with `UndefinedFunctionError`

**R3. SecondBrain.Indexer DB connection race**
- Task.Supervised processes lose connection ownership mid-transaction
- 3 test failures, likely manifests in production as intermittent indexing failures

**R4. Three harvesters declared but not implemented**
- `@valid_harvesters ~w(git session vault usage brain_dump)` in run.ex
- Only `git_harvester.ex` and `session_harvester.ex` exist
- Attempting to start vault/usage/brain_dump harvesters → `UndefinedFunctionError`

### YELLOW — Orphaned / Incomplete

**Y1. Seven frontend stores join non-existent channels**
- `token-monitor-store` → "intelligence:tokens"
- `vm-health-store` → "intelligence:vm"
- `jarvis-store` → "jarvis:lobby"
- `prompt-workshop-store` → "prompts:lobby"
- `decision-store` → "decisions:lobby"
- `pipeline-store` → "pipeline:lobby"
- `project-graph-store` → "project_graph:lobby"
- Each gets a Phoenix join error on mount. Silent failures, but clutters logs and confuses debugging.

**Y2. Approved proposals dispatch nowhere**
- `Proposals.approve_proposal/1` broadcasts "proposal_approved"
- No consumer creates an Execution or dispatches to an agent
- The intended core loop (brain dump → intent → proposal → execution → harvest) is broken at the proposal→execution boundary

**Y3. Stream.Manager has stub `maybe_record_activity/2`**
- Replaced real `Monitor.record_activity` with a no-op
- Babysitter incidents aren't actually being recorded

**Y4. SpaceController defers org fetching**
- `Ema.Orgs.Org` module doesn't exist yet
- TODO comment in controller, returns empty org data

**Y5. Four different vault path resolution methods**
- `compile_env` (baked at compile), `get_env` (runtime), `System.get_env()`, hardcoded
- Changing vault path at runtime won't apply uniformly

**Y6. Notes channel possibly orphaned**
- `notes_channel.ex` joins "notes:updates" but no frontend store connects to it

## 9. The Intended Core Loop vs Reality

**Design intent** (from `.superman/context.md`):
```
brain dump item
  → intent folder
  → Execution (DB + runtime)
  → proposal/approval
  → agent delegation (structured packet)
  → harvested result
  → patch intent files
  → next execution
```

**What actually works:**
```
brain dump item → PubSub broadcast                    ✅
proposal engine (generate → refine → debate → tag)    ✅
approve proposal → broadcast "proposal_approved"       ✅
                → ??? (nothing consumes this)          ❌
execution dispatch                                     ❌ (framework exists, no wiring)
agent delegation                                       ❌ (no structured packet format)
result harvesting                                      ❌ (session harvester not implemented)
intent patching                                        ❌ (not started)
```

**The loop is ~40% complete.** The biggest gap is the proposal→execution→agent handoff.
