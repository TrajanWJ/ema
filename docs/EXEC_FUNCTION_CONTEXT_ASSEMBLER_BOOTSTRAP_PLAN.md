# EMA Executive Function + Context Assembler Bootstrap Plan

Generated: 2026-04-07
Status: active working plan

## Mission

Build EMA into a shared workspace and executive-functioning system for humans and agents by:
1. unifying runtime truth, intent schematic, memory, wiki, loops, and semantic expansion under a single context-assembler contract
2. cross-pollinating high-signal knowledge from OpenClaw, operator docs/preferences, research, and prior semantic-system ideas
3. seeding EMA with durable bootstrap corpus and packet-ready context products

## Track A — Internal spine

### A1. Packet contract
- define packet families: operator, project, intent, execution, briefing, wiki overlay
- define common envelope fields and authority metadata
- define packet resolver inputs: mode, scope, actor, session, project, intent

### A2. Source adapters
- runtime adapter
- intent schematic adapter
- memory adapter
- loops adapter
- wiki adapter
- semantic expansion adapter
- session continuity adapter

### A3. Ranking + safety
- runtime-first authority for operational questions
- hybrid rules + scoring
- sensitivity labels: public/internal/private/secret
- stale/drift/contradiction surfacing

### A4. Surface consumers
- refactor Brief onto packet model
- align context injection with packet model
- add query-conditioned wiki overlays
- prepare CLI/MCP/OpenClaw-facing consumers

### A5. Runtime authority diagnostics
- reconcile service-manager truth, process truth, and API truth
- expose mismatch as first-class runtime context

## Track B — Cross-pollination and seed corpus

### B1. Source harvest
Gather and classify from:
- OpenClaw workspace and memory
- operator docs/preferences
- EMA host docs/code/runtime state
- research artifacts and registries
- prior semantic/memory-system patterns

### B2. Source typing
For each imported item record:
- source
- kind
- confidence
- freshness
- applicability
- durability
- sensitivity

### B3. Seed corpus families
- operator profile
- workflow patterns
- approval/safety norms
- memory conventions
- context assembly patterns
- research distillations
- anti-patterns
- packet seeds
- wiki/bootstrap pages
- intent/bootstrap pages

### B4. Promotion rules
- what becomes durable memory
- what stays advisory/reference only
- what can become packet-ready by default
- what remains quarantined pending review

## Immediate implementation order

1. context assembler packet/API design
2. runtime authority diagnostics slice
3. memory/loops/schematic adapters

## Working principle

Do not dump everything into prompts or docs. Normalize, classify, rank, and package it so EMA can share continuity and executive function cleanly across humans and agents.
