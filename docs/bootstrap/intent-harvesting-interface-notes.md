# Intent Harvesting Interface Notes

Updated: 2026-04-07

## Working preference captured

Trajan prefers structured, composable intention-expression systems over vague freeform clarification loops.

A particularly strong preference is for multi-axis labeled choice systems such as:
- A/B/C/D dimensions
- 1/2/3 variants per dimension
- mix-and-match blended profiles

This should be treated as a durable operator preference for EMA and related agent systems.

## Current favored interface concept

### Intent interface idea: Matrix + Sliders

Intent should be expressed as:
1. **Shape** — structural selection via matrix
2. **Tuning** — behavioral nuance via sliders

### Shape / Matrix layer
Use a compact multi-axis chooser for things like:
- objective
- scope
- autonomy
- output shape
- time horizon
- risk posture

Example:
- `A2 B3 C1 D2`

### Tuning / Slider layer
Use lightweight sliders for things like:
- depth
- rigor
- speed
- creativity
- initiative
- risk
- verbosity

Example:
- `depth 8`
- `rigor 9`
- `initiative 7`
- `risk 2`

Or compressed:
- `D8 R9 I7 K2`

## Core principle

Intent = **Shape + Tuning**

This is a strong candidate for EMA-native intent elicitation because it combines:
- high agency
- composability
- machine parseability
- fast convergence
- nuanced control without long prose
